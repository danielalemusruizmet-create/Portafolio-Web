⚖️ Tema ético: Transparencia algorítmica y sesgo en datos de entrenamiento

Taller práctico con Teachable Machine · Análisis de sesgos en imágenes de laboratorio.

Herramienta principal: Teachable Machine de Google — accede en teachablemachine.withgoogle.com · No necesitas cuenta, funciona en el navegador.

🎯 ¿Qué vas a hacer?

Entrenarás un modelo de reconocimiento de imágenes para clasificar objetos físicos de laboratorio (ej. prisma, lente, imán). Pero con un giro: una clase estará intencionalmente sesgada — solo imágenes tomadas bajo ciertas condiciones (mismo ángulo, misma iluminación). Luego comprobarás cómo esto afecta la predicción.

📌 Paso a paso

  • Paso 1: Abre Teachable Machine → selecciona "Image Project" → "Standard image model"
  • Paso 2: Crea 3 clases: Clase A (prisma), Clase B (imán), Clase C (lente)
  • Paso 3: Para Clase A: toma 30+ fotos con cámara frontal, siempre mismo ángulo y fondo blanco
  • Paso 4: Para Clase B y C: toma fotos desde ángulos variados, distintos fondos, distintas iluminaciones
  • Paso 5: Entrena el modelo (botón "Train Model") — espera hasta que el indicador llegue a 100%
  • Paso 6: Prueba el modelo: muestra el objeto de Clase A desde ángulos distintos a los de entrenamiento. ¿Lo reconoce?
  • Paso 7: Exporta tu modelo (Export Model → "Download my model") y guarda el link
  • Paso 8: Completa la pestaña "Mi reflexión"

💡 Preguntas orientadoras mientras trabajas

• ¿Por qué el modelo entrenado solo con un ángulo falla cuando el ángulo cambia?

• ¿Qué pasaría si entrenáramos un modelo de reconocimiento facial solo con fotos de personas de un grupo étnico?

• ¿Quién es responsable de los errores del modelo: el algoritmo, los datos, o quien diseñó el experimento?
